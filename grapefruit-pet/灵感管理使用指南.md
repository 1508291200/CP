### 输入技巧

#### 1. 使用 #标签
在内容中输入 `#标签名`，系统会自动识别：
```
今天学习了 #React #Hooks，感觉很有用
```
自动识别标签：`React`、`Hooks`

#### 2. 自动分类
系统会根据内容自动分类：
- **技术**：代码、算法、架构、优化等关键词
- **项目**：开发、功能、产品、实现等
- **学习**：学习、笔记、教程等
- **工作**：会议、任务、deadline等
- **设计**：UI、UX、界面、配色等
- **创意**：创意、点子、灵感等
- **生活**：日常、心情、感悟等
- **商业**：营销、盈利、推广等

#### 3. 技术关键词识别
系统自动识别常见技术关键词：
- 编程语言：`JavaScript`、`Python`、`TypeScript`
- 框架：`React`、`Vue`、`Node.js`、`Electron`
- 概念：`AI`、`ML`、`前端`、`后端`、`算法`

### 键盘快捷键

**快速输入框内：**
- `Ctrl+Enter` - 保存
- `Esc` - 取消
- `Tab` - 切换输入框

## 灵感库

### 打开灵感库

**两种方式：**

1. 右键菜单 → `💡 灵感管理` → `📚 浏览灵感库`
2. 右键菜单 → `💡 灵感管理` → `🔍 搜索灵感`

### 灵感库界面

```
┌─────────────────────────────────────┐

├─────────────────────────────────────┤

│  └───────────┘  └───────────┘      │
└─────────────────────────────────────┘
```

### 灵感卡片

每张卡片显示：
- **标题**：灵感的标题（如果有）
- **内容摘要**：最多显示3行
- **标签**：自动识别的标签
- **分类**：自动分类结果
- **日期**：创建时间（今天、昨天、X天前）
- **状态图标**：
  - `⭐` 已收藏 / `☆` 未收藏
  - `✅` 已实现 / `⭕` 未实现

### 卡片操作

**收藏/取消收藏**：
- 点击卡片右上角的星星图标
- 收藏的灵感会优先显示

**标记已实现**：
- 点击卡片底部的圆圈图标
- 已实现的灵感会显示 `✅`

## 最佳实践

### 1. 标题命名建议

- ✅ **好的标题**：
  - "React性能优化方案"
  - "登录流程改进想法"
  - "配色灵感：紫色渐变"

- ❌ **避免**：
  - "想法"（太笼统）
  - "记录一下"（无意义）

### 2. 内容记录建议

- ✅ **简洁明了**：记录核心想法，不要长篇大论
- ✅ **使用标签**：方便后续搜索和分类
- ✅ **关键词明确**：帮助系统准确分类

### 3. 定期回顾

- 📅 每周回顾一次灵感库
- ⭐ 标记重要灵感为收藏
- ✅ 实现后及时标记
- 🗑️ 删除过时的想法（后续支持）

## 技术特性

### 智能分类算法

系统使用多层级分类策略：
- **关键词匹配**（70%权重）：分析内容中的关键词
- **标签分析**（30%权重）：基于标签推断分类
- **综合评分**：选择得分最高的分类

### 标签提取

自动提取以下标签：
1. `#标签` 格式的显式标签
2. 技术关键词（React、Python等）
3. 常见概念词汇

### 数据结构

```typescript
{
  id: "唯一ID",
  title: "标题（可选）",
  content: "内容",

  tags: ["标签1", "标签2"],
  favorite: false,      // 是否收藏
  implemented: false,   // 是否已实现
  createdAt: 时间戳,
  updatedAt: 时间戳
}
```

## 常见问题

### Q: 灵感数据会丢失吗？
**A:** 不会。所有数据自动持久化存储在本地，关闭应用也不会丢失。

### Q: 可以修改已保存的灵感吗？
**A:** 目前版本支持收藏和标记已实现，编辑功能将在后续版本支持。

### Q: 能导出灵感数据吗？
**A:** 导出功能（JSON/Markdown/CSV）将在后续版本支持。

### Q: 如何删除不需要的灵感？
**A:** 删除功能将在后续版本支持，目前可以不收藏来降低优先级。

### Q: 自动分类不准确怎么办？
**A:** 系统会不断学习优化，手动分类功能将在后续版本支持。
