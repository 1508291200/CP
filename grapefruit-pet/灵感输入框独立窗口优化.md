# 灵感输入框独立窗口优化

## 🎯 问题解决

### 之前的问题
- 灵感输入框在React组件内渲染
- 受主窗口限制（100x120px）
- 1400px的输入框无法完全显示
- 被小窗口裁切

### 解决方案
采用和菜单相同的方式：**创建独立的BrowserWindow**

---

## 🏗️ 技术实现

### 1. 创建独立窗口

在 `electron/main.ts` 中创建新的窗口函数：

```typescript
let inspirationWindow: BrowserWindow | null = null;

function createInspirationWindow() {
  if (inspirationWindow) {
    inspirationWindow.focus();
    return;
  }

  const windowWidth = 1400;
  const windowHeight = 700;
  
  // 居中显示
  const x = Math.round((screenWidth - windowWidth) / 2);
  const y = Math.round((screenHeight - windowHeight) / 2);

  inspirationWindow = new BrowserWindow({
    width: windowWidth,
    height: windowHeight,
    x, y,
    frame: false,
    transparent: true,
    resizable: false,
    alwaysOnTop: true,
    skipTaskbar: true,
    webPreferences: {
      nodeIntegration: true,
      contextIsolation: false,
    },
  });

  inspirationWindow.loadFile('inspiration-input.html');
  
  inspirationWindow.once('ready-to-show', () => {
    inspirationWindow?.show();
  });
}
```

### 2. 创建独立HTML页面

`inspiration-input.html` - 完整的灵感输入界面：
- 1400x700 尺寸
- 完整的表单和样式
- 内置JavaScript逻辑
- 独立运行，不依赖React

### 3. IPC通信流程

```
用户点击菜单
  ↓
createInspirationWindow() - 创建独立窗口
  ↓
加载 inspiration-input.html
  ↓
用户填写并保存
  ↓
ipcRenderer.send('inspiration:save', data)
  ↓
主进程转发给主窗口
  ↓
mainWindow.webContents.send('inspiration:saveData', data)
  ↓
App.tsx 接收并保存到Store
```

## 🔄 数据流

### 保存流程

1. **用户操作**
   ```html
   用户在独立窗口填写内容 → 点击保存按钮
   ```

2. **窗口内处理**
   ```javascript
   // inspiration-input.html
   btnSave.addEventListener('click', () => {
     ipcRenderer.send('inspiration:save', {
       title, content, tags, category
     });
     window.close();
   });
   ```

3. **主进程转发**
   ```typescript
   // electron/main.ts
   ipcMain.on('inspiration:save', (_, data) => {
     mainWindow.webContents.send('inspiration:saveData', data);
   });
   ```

4. **渲染进程保存**
   ```typescript
   // App.tsx
   window.electronAPI.onSaveInspirationData((data) => {
     const { addInspiration } = useInspirationStore.getState();
     addInspiration(data);
   });
   ```

## 🔧 开发细节

### 为什么使用独立HTML而不是React组件？

1. **简单直接**
   - 无需路由配置
   - 无需额外打包
   - 加载速度快

2. **独立运行**
   - 不依赖主窗口
   - 不占用主应用资源
   - 更好的隔离性

3. **完全控制**
   - 直接控制窗口大小
   - 不受CSS限制
   - 精确的样式控制

### nodeIntegration vs contextIsolation

```typescript
webPreferences: {
  nodeIntegration: true,      // 允许使用Node.js API
  contextIsolation: false,    // 简化IPC通信
}
```

**为什么这样设置**：
- 独立窗口，安全性可控
- 简化代码，直接使用require('electron')
- 不需要preload桥接
